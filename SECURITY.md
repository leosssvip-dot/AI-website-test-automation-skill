# Security Policy

## Supported Version

The `main` branch is the current supported version until tagged releases are introduced.

## Reporting A Vulnerability

Please report security issues through GitHub Security Advisories when available. If advisories are not enabled, open a GitHub issue with a minimal, non-sensitive description and avoid posting secrets, tokens, cookies, customer data, raw provider payloads, screenshots with PII, or exploit details.

## Testing Boundaries

This project helps agents plan and automate website testing. Only test systems you own or are explicitly authorized to test. Intrusive security testing, fuzzing, credential attacks, data exfiltration, and provider-cost-incurring live tests require explicit authorization, a defined scope, cost limits where relevant, and redacted evidence.
